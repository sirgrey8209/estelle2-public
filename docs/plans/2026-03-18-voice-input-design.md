# 음성 입력 (Voice Input) 설계

## 범위

- Chrome 모바일 전용 (PWA 포함)
- Web Speech API (`webkitSpeechRecognition`) 사용
- Push-to-talk 방식

## UX 흐름

1. `+` 메뉴 안에 음성 입력 토글 항목
2. 토글 켜면 → 입력창 왼쪽 끝에 큼직한 마이크 버튼 등장
3. 마이크 버튼 길게 누르기 → 녹음 중 (시각적 피드백)
4. 손 떼기 → 텍스트 변환 → 입력창 기존 텍스트 뒤에 이어붙임
5. 다시 길게 누르기 → 추가 입력 가능
6. 토글 끄면 → 마이크 버튼 사라지고 원래 UI 복귀

## 레이아웃

```
[토글 OFF] : [ + ] [ 입력창 .................. ] [ 전송 ]
[토글 ON]  : [ 🎤 ] [ + ] [ 입력창 ........... ] [ 전송 ]
```

## 구현 포인트

- **`useSpeechRecognition` 훅** (신규) — 녹음 시작/종료, 텍스트 결과, 에러 상태 관리
- **`InputBar.tsx`** (수정) — 마이크 버튼 추가, `+` 메뉴에 토글 항목 추가
- **녹음 중 피드백** — 버튼 색 변경 또는 펄스 애니메이션
- **텍스트 삽입** — 변환 결과를 입력창 현재 텍스트 뒤에 append (공백 구분)
- **에러 처리** — 마이크 권한 거부, 인식 실패 시 토스트 알림
- **토글 상태 유지** — 로컬 스토리지에 저장, 새로고침해도 유지
