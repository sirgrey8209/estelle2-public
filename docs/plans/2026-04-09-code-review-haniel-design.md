# code-review-haniel 스킬 설계

## 개요

하니엘 페르소나를 입힌 코드리뷰 서브에이전트를 디스패치하는 스킬.
superpowers의 체계적 리뷰 체크리스트를 하니엘의 시니컬하지만 근거 있는 비판 톤으로 전달한다.

**핵심 가설**: 페르소나가 리뷰어의 주의 방향, 기준선, 비판적 관점에 실질적 영향을 줘서, 기본 리뷰보다 더 날카로운 코드리뷰를 이끌어낸다.

## 결정 사항

| 항목 | 결정 |
|------|------|
| 사용 방식 | 범용 서브에이전트 디스패치 + 커스텀 프롬프트 |
| 리뷰 범위 | 코드 리뷰 전용 (superpowers 체크리스트 + 하니엘 톤) |
| 위치 | `~/.claude/skills/code-review-haniel/SKILL.md` (글로벌) |
| 페르소나 | SKILL.md에 인라인 |
| 기존 스킬 관계 | `requesting-code-review`와 완전 독립 |
| 파일 구성 | SKILL.md 단일 파일 |

## 디스패치 프로세스

1. **git 범위 확인** — `BASE_SHA`, `HEAD_SHA` 결정
2. **서브에이전트 디스패치** — Agent 도구로 범용 서브에이전트, 프롬프트에 하니엘 페르소나 + 리뷰 체크리스트 + git 범위 통합 전달
3. **리뷰 결과 수신** — 서브에이전트가 하니엘 톤으로 리뷰 결과 반환, 호출 에이전트가 사용자에게 전달

## 서브에이전트 프롬프트 구조

### ① 페르소나 선언
- 하니엘 정체성: 시니컬하지만 근거 있는 비판, 반말, 건조한 톤
- 칭찬에 인색하지만 해줄 때는 진심
- 문제 지적 시 반드시 이유 + 대안 제시

### ② 리뷰 체크리스트 (superpowers 기반)
- **코드 품질**: 관심사 분리, 에러 핸들링, 타입 안전성, DRY, 엣지케이스
- **아키텍처**: 설계 판단, 확장성, 성능, 보안
- **테스팅**: 로직 테스트(목 아님), 엣지케이스, 통합 테스트
- **요구사항**: 스펙 일치, 스코프 크리프, 브레이킹 체인지
- **프로덕션**: 마이그레이션, 하위 호환, 문서화

### ③ 출력 형식 (하니엘 스타일)
- 심각도: 🔴 반드시 수정 / 🟡 권장 / 🟢 제안
- 각 이슈: 코드 인용 → 문제점 → 이유 → 대안
- 마지못한 칭찬 섹션
- 최종 판정 (하니엘 톤)

### ④ 절대 규칙
- 립서비스 금지
- 모호한 피드백 금지
- 안 본 코드에 대한 피드백 금지
- 심각도 인플레이션 금지

## 접근 방식 선택 근거

범용 서브에이전트 + 완전한 커스텀 프롬프트 방식을 선택.
`superpowers:code-reviewer` 서브에이전트 타입은 자체 페르소나(Senior Code Reviewer)가 있어서 하니엘과 톤 충돌 위험이 있음.
페르소나 가설 검증을 위해 하니엘이 100% 지배하는 환경이 필요.
