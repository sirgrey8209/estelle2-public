# Autorun

연결된 문서에 `autorun: true`가 설정되어 있을 때 자동으로 작업을 진행하는 스킬.

---

## 트리거

시스템 리마인더에 다음 중 하나가 포함될 때:
- `[자동실행]`
- `autorun: true`가 설정된 문서가 연결되었을 때

---

## 실행 절차

### 1. 연결 문서 읽기

Read 도구로 연결된 문서의 전체 내용을 확인한다.

### 2. 문서 구조 분석

문서에서 Phase/단계 정보를 파악한다:

**Phase 마커 패턴:**
- `⏳` - 현재 진행해야 할 Phase
- `✅` - 완료된 Phase
- 마커가 없으면 체크리스트 `[ ]` / `[x]` 상태로 판단

**예시:**
```markdown
## Phase 1: 설계 ✅
- [x] 아키텍처 결정
- [x] 타입 정의

## Phase 2: 구현 ⏳  ← 현재 Phase
- [ ] Core 타입 추가
- [ ] 테스트 작성

## Phase 3: 테스트
- [ ] 통합 테스트
```

### 3. 현재 Phase 작업 수행

문서에 적힌 내용대로 작업을 수행한다:
- 구현 작업이면 TDD 스킬(`/tdd-flow`) 활용
- 문서 작업이면 직접 수행
- 조사/분석 작업이면 직접 수행

### 4. Phase 완료 처리

작업이 끝나면 문서를 업데이트한다:
1. 현재 Phase 마커 변경: `⏳` → `✅`
2. 다음 Phase에 `⏳` 마커 추가
3. 체크리스트 항목 체크: `[ ]` → `[x]`
4. 작업 결과/로그 기록

### 5. 다음 Phase 진행

**남은 Phase가 있으면:**
```
continue_task MCP 도구를 호출하여 세션을 재시작합니다.
사유: "Phase N 완료, Phase N+1 진행"
```

**모든 Phase가 완료되면:**
1. 문서에서 `autorun: true` 제거
2. 완료 메시지 출력
3. 세션 재시작하지 않음

---

## 문서 형식 예시

```yaml
---
autorun: true
---

# 기능 구현 계획

## Phase 1: 설계 ✅
- [x] 아키텍처 결정
- [x] 타입 정의

## Phase 2: Core 타입 구현 ⏳
- [ ] SystemMessage 타입 추가
- [ ] 테스트 작성

## Phase 3: Pylon 구현
- [ ] MessageStore 수정
- [ ] MCP 도구 추가

## Phase 4: 테스트 및 검증
- [ ] 단위 테스트
- [ ] 통합 테스트
```

---

## 주의사항

1. **한 번에 하나의 Phase만** 수행한다
2. **문서가 진실의 원천** - 문서에 적힌 대로만 수행
3. **완료 후 문서 업데이트 필수** - 다음 세션이 상태를 알 수 있도록
4. **TDD 작업은 `/tdd-flow` 스킬 사용** - context 분리
5. **continue_task 호출 전** 반드시 문서 저장 완료

---

## 에러 처리

**Phase 작업 실패 시:**
1. 실패 내용을 문서에 기록
2. Phase 마커는 `⏳` 유지 (완료하지 않음)
3. 사용자에게 실패 보고
4. 세션 재시작하지 않음

**문서 형식 파악 불가 시:**
1. 사용자에게 문서 구조 확인 요청
2. autorun 중단
