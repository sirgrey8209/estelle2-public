# Estelle

클로드 코드를 원격으로 제어하고 대화형으로 사용하게 하는 멀티디바이스 동기화 시스템이에요.

## 핵심 구조

- **Pylon** — 상태 관리 + Claude Agent SDK 실행. 모든 대화/파일/세션 상태의 단일 진실 원천
- **Relay** — 순수 라우터. 클라이언트와 Pylon 사이 WebSocket 메시지 중계
- **Client** — React PWA. 모바일/데스크톱에서 대화 UI 제공

## 동작 방식

사용자가 Client에서 메시지를 보내면 → Relay가 Pylon으로 중계 → Pylon이 Claude Agent SDK로 실행 → 결과를 실시간 스트리밍으로 모든 연결된 디바이스에 동기화.

## 알아둘 것

- **에스텔 허브** — 프로젝트들을 등록하고 웹에서 확인할 수 있는 대시보드. 상세는 `estelle-hub` 스킬 참고.
- **문서 링크** — 대화에 외부 문서를 연결해서 컨텍스트로 활용하는 기능이 있음.
- **아카이브** — 클라우드 서버에 공유 문서 저장소. 두 PC 간 자료 공유용. MCP 도구 7개 (`archive_write/read/list/glob/grep/delete/rename`)로 접근. 허브 UI에 VSCode 스타일 뷰어 통합. 저장 경로: `<아카이브 경로 - 유저 환경에 맞게 설정>`. HTTP API: 포트 3009 (Caddy 프록시 `/archive/*`).
- **사용자 환경** — 유저는 원격 PC나 모바일에서 접속하고 있을 수 있음. 로컬 파일시스템 접근이 불가능한 환경을 고려할 것.

## 워크스페이스

- 경로: `<워크스페이스 경로 - 유저 환경에 맞게 설정>`
- 서버: `<유저 도메인 - 유저 환경에 맞게 설정>` (<서버 IP - 유저 환경에 맞게 설정>)

## 관련 스킬

- `estelle-hub` — 허브에 프로젝트 등록/배포
- `estelle-master` — 개발용 상세 코드 레퍼런스 (에스텔 워크스페이스 로컬)
